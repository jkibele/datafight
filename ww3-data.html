Getting WW3 Wave Data and Messing with it

Wed 23 October 2013

A colleague here at the lab is concerned about how some cyclone generated swell events might have affected her experements on the reef. She asked me to get some wave data and find out when the biggest swell events were within the time period of December 2012 to March 2013 were. Here's what I did with python to try and find an answer.

In [61]: 
import netCDF4 as ncdf
import pandas as pd

I used the marinexplore website to extract some NOAA WaveWatch III data. There's a way to get this directly from NOAA but I haven't figured it out yet. I did a little looking into it but it seems that several key websites are down due to the utterly silly federal government shutdown of 2013.

Once you've prepared a dataset on marinexplore, you can get a DAP url for the data. You can use that URL with the netCDF4 library to access your data:

In order to access the right data, you have to translate geographic coordinates into data coordinates.

In [64]: 
def coord(latitude,longitude,ncf):
    """
    For a given lat and lon, return the matrix coordinates needed to get that grid cell.
    """
    latcoord = np.where(ncf.variables['lat0'][:]==latitude)[0][0]
    loncoord = np.where(ncf.variables['lon0'][:]==longitude)[0][0]
    return latcoord,loncoord

The dates are in units of "seconds since 1970-01-01 00:00:00 UTC" so I'll want to translate those into python datetime objects. I will also put the data I'm interest in into some variables for easy access. I probably should switch the datetimes to the NZ timezone but I can't be bothered right now.

In [65]: 
dts = ncdf.num2date(wwds.variables['time0'][:],wwds.variables['time0'].units)
mylat,mylon = coord(-36.0,175.5,wwds)

As I understand it (not very well), you need to think about more than just wave height to get a measure of how "wave exposed" something is. Wave energy includes period and significant wave height. It seems a better measure to use. I looked up how to calculate wave energy on the internets. I found this, this, and this. I initially thought that looked pretty easy to comprehend. Then I realized that the first two links were talking about energy density or energy per unit of area. That takes out the wave period component (I guess) so I don't think that's really what I want. The third link seems to actually refer to the quantity I want to calculate but it isn't very explicit so I looked around a bit more. I seems like what I really want is "wave power". Wikipedia describes that here and I found a painfully detailed PDF here.

First, just import some stuff to get IPython ready to do symbolic math.

Set up the wave power formula according to the wikipedia page. This also seems to be pretty much what the painfully detailed PDF mentioned above says on page 13. The formula is:

\(P = \frac{\rho g^2}{64 \pi} H_{m0}^2 T\)

Where:

\(P\) = wave power
\(\rho\) = density of seawater = \(1.025 g/ml\)
\(g\) = acceleration by gravity = \(9.80665 m/s^2\)
\(H_m0\) = significant wave height
\(T\) = wave period (\(s\))

When set up as a sympy symbolic expression, it looks like this:

In [70]: 
g,H,T = symbols('g H_m0 T')
wavePower = ( (rho * g**2)/(64*pi) ) * H**2 * T
wavePower
Out[70]:
$$\frac{H_{m0}^{2} T g^{2} \rho}{64 \pi}$$

Now, I will substitute in the values for \(\rho\) and \(g\) and make a function of \(H_m0\) and \(T\) that returns wave power in \(\frac{kW}{m}\).

In [71]: 
f = lambdify( (H,T), wavePower.subs({rho:1.025,g:9.80665}) )

Try out the function with the values use here and see if we get the same answer of \(36 \frac{kW}{m}\).

In [72]: 
f(3,8)
Out[72]:
$$35.2994441147$$

The answer is close but not exactly the same. The reason it's not the same is that they approximated \(\frac{\rho g^2}{64 \pi}\) as 0.5. Sympy calculated it for us to a nutty degree of precision. Doing the following shows our approximation out to 5 places:

In [73]: 
wavePower.subs({rho:1.025,g:9.80665}).evalf(5)
Out[73]:
$$0.49027 H_{m0}^{2} T$$

Now I will use the wave power function (f) that I just created to build another column in my WW3 dataframe. The new column will have the wave power calculated for each day of the dataset.

In [74]: 
ww3_df['power'] = pd.Series( f(ww3_df.height,ww3_df.period), index=ww3_df.index )

In this case the coversion to power doesn't appear to have changed the order of the biggest events. The swell at the end of December was still the biggest, early January still the second biggest, etc. The conversion does show how much more energy there was in that late December swell compared to the normal "non-storm" state of affairs.

And just to confuse things more all of these metrics can change depending on the sampling rate. The daily wave power peak (23 Dec 2012) might not be the peak if we were looking for a "per storm" power total or a "per week" power total. I'll resample the data in a coulple of different ways to demonstrate:

In [81]: 
twoday = ww3_df.resample('2D').power.resample('D',fill_method='ffill')
fourday = ww3_df.resample('4D').power.resample('D',fill_method='ffill')
oneweek = ww3_df.resample('7D').power.resample('D',fill_method='ffill')
tenday = ww3_df.resample('10D').power.resample('D',fill_method='ffill')
rsdf = pd.DataFrame( {'1 day':ww3_df.power,'2 day':twoday,'4 day':fourday, \
                      '7 day':oneweek,'10 day':tenday}, index=ww3_df.index )
rsdf.plot(figsize=(10,8),x_compat=True)

Anyway, you can see that resampling changes the order of biggest event. The late Jan / early Feb storm had more power overall. So it all kind of depends on exactly what you want to measure I guess.
